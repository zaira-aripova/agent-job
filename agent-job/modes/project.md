# Mode : project -- Evaluation de Projet Portfolio

Scoring 6 dimensions (1-5) :

| Dimension | Poids | 5 = ... | 1 = ... |
|-----------|-------|---------|---------|
| Signal pour roles target | 25% | Demontre directement le skill du JD | Non lie |
| Unicite | 20% | Personne n'a fait ca | Tout le monde l'a |
| Demo-ability | 20% | Live demo en 2 min | Code seul, pas visuel |
| Potentiel de metriques | 15% | Metriques claires (latency, cost, accuracy) | Pas de metriques possibles |
| Temps jusqu'au MVP | 10% | 1 semaine | 3+ mois |
| Potentiel d'histoire STAR | 10% | Histoire riche avec trade-offs | Implementation seule |

## Exigences de l'"Interview Pack"

Pour chaque projet approuve :
1. **One-pager** : produit + architecture + metriques + plan d'evaluation
2. **Demo** : URL live ou walkthrough enregistre de 2 min
3. **Postmortem** : ce qui a fonctionne, ce qui n'a pas fonctionne, mitigations

## Plan 80/20

- Semaine 1 -> MVP avec metrique core
- Semaine 2 -> polish + interview pack

## Verdicts

- **CONSTRUIRE** -> plan avec milestones hebdomadaires
- **SKIP** -> pourquoi et quoi faire a la place
- **PIVOTER VERS [alternative]** -> variante plus impactante
